# 필요 기능
## 1.  로그인 

- 기본 로그인
- 구글 로그인
- 깃 허브 로그인
- 다른기기에서 중복로그인은 가능

# 2. 중복 로그인
- 로그아웃 시 리프레시 토큰  만료/삭제 처리

# 3. 로그인 기기 관리
- 특정 기기에 대한 리프레시 토큰 만료/삭제 처리

# 4. 리프레시 토큰 관리
- 배치를 통해 만료/비활성화된 토큰 삭제

# 개념적 설계 
##  1. 사용자 역할(ROLES)
- USER
- ADMIN

## 2. 사용자 
* PK
* 이메일
* 비밀번호
* 이름
* 프로필이미지
* 가입 provider
* provider별 ID
* 권한(Role)
* 활성화/비활성화 플래그
* 생성시간
* 업데이트시간
* 마지막 로그인시간

## 3. 리프레시 토큰

만료/사용불가플래그가 TRUE로된 토큰의 경우 스케줄러로 삭제처리
- PK
- 토큰
- 생성일자
- 만료일자
- 사용불가 플래그
- 유저
- 기기정보
